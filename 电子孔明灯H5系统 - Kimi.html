电子孔明灯H5系统

我想生成一个电子孔明灯的H5系统 主要用作《一屋暗灯》广播剧的互动活动 核心逻辑（完全对标晚安短信） 用户报名留手机号 → 每晚 22:00 自动收到一条孔明灯许愿短信 短信内容 = 角色晚安语 + 剧中救赎台词 + 孔明灯意象 同步在 H5 / 小程序生成电子孔明灯，可查看、分享 完结日统一放飞 “全服孔明灯”，做情感收官

我来为你设计这个《一屋暗灯》广播剧电子孔明灯H5系统。这是一个很有情感温度的互动活动，对标"晚安短信"的仪式感，结合广播剧剧情做沉浸式体验。

系统架构设计

1. 核心功能模块

表格
模块功能技术方案
报名系统手机号收集 + 角色偏好选择微信授权 + 短信验证
定时短信每晚22:00推送第三方短信服务商定时任务
电子孔明灯生成、查看、分享Canvas/WebGL + 社交分享API
完结放飞全服孔明灯动画实时数据可视化 + 粒子特效

2. 短信内容结构设计

plain
复制
【一屋暗灯】
"今晚的月亮很亮，像那天你带我回家时的路灯。"

——宋星阑

🔮 你的孔明灯已升空，点击查看今晚的星空
[短链跳转H5]
内容库设计（按剧情节点编排）：
  • Week 1（破碎期）：角色独白，剧中虐心台词，孔明灯"未点燃"状态
  • Week 2（纠缠期）：双向视角台词，灯芯渐亮
  • Week 3（救赎期）：高光救赎台词，灯火通明
  • 完结日：全员放飞，"天灯满城"

3. 电子孔明灯视觉系统

孔明灯状态机：
plain
复制
未点燃 → 点燃中 → 升空 → 星空常驻
(灰白)   (暖黄)    (橙红渐隐)  (微光星点)
交互设计：
  • 每日首次打开：孔明灯从"点燃"动画开始
  • 历史回顾：横向滑动查看已收到的灯（时间轴）
  • 分享卡片：自动生成带角色台词的海报图

4. 技术实现要点

前端（H5）：
  • 孔明灯动画：Canvas 2D/轻量WebGL，兼容低端机
  • 分享生成：html2canvas 服务端渲染避免跨域
  • 适配：微信内置浏览器 + 普通浏览器双兼容
后端：
  • 短信队列：Redis + Celery/Node-cron 定时任务
  • 用户分群：按报名角色偏好分组推送不同内容
  • 防刷机制：手机号频率限制 + 微信OpenID绑定
数据埋点：
  • 短信打开率、H5停留时长、分享率、完结日回流率

5. 完结日"全服放飞"特效

技术方案：
  • WebSocket实时连接，显示当前在线用户数
  • 每盏灯 = 一个用户数据点，带角色色标
  • 粒子系统模拟万灯齐飞，镜头缓缓拉升俯瞰视角
  • 背景音乐：广播剧主题曲片段

开发建议

MVP版本优先级：
  1. 报名页 + 短信接入（测试期间可手动触发）
  2. 单盏孔明灯生成 + 分享
  3. 定时任务稳定运行
  4. 完结日特效（可提前录制视频兜底）
需要确认的细节：
  • 活动周期多长？（建议21天，3周剧情节奏）
  • 是否区分角色线？（宋星阑/另一主角双视角推送）
  • 短信服务商有偏好吗？（推荐阿里云/腾讯云，到达率高）
如果你需要，我可以继续输出：
  • 完整的页面原型交互说明
  • 短信内容库的具体文案（按剧情节点编排）
  • 技术选型对比表（自研 vs 低代码平台）
这个项目的情感设计很细腻，孔明灯作为"未说出口的救赎"的意象，和《一屋暗灯》的骨科救赎主题高度契合。
